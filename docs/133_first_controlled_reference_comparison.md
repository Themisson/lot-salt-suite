## Reference Contract

The fixture uses the axisymmetric elastic diagnostic relation:

```text
sigma_theta_initial = far_field_stress_compression_positive_Pa
sigma_theta_current = sigma_theta_initial - wellbore_pressure_Pa
tensile_condition = -sigma_theta_current
fracture_margin = tensile_condition - tensile_strength_Pa
fracture_initiated = fracture_margin >= 0
```

The expected values are written directly in
`tests/fixtures/comparison/phase_first_controlled_reference/axisymmetric_reference_cases.json`.
They are not generated by the runtime provider.

## Cases

| Case | Purpose |
|---|---|
| `compressive_not_reached` | Hoop stress remains compressive. |
| `zero_hoop_not_reached` | Hoop stress is zero but below tensile strength. |
| `tension_below_strength_not_reached` | Tensile condition exists but remains below strength. |
| `tension_above_strength_reached` | Tensile condition exceeds strength. |
| `exact_threshold_reached` | Threshold behavior uses `>=`. |
| `high_pressure_reached` | High pressure produces clear reach. |
| `low_pressure_compressive_safe` | Low pressure remains safely compressive. |
